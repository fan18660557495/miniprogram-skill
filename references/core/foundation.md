# 通用基础规范

这个文件定义小程序项目里长期稳定、跨项目通用的基础规则。

## 语言规则

- 默认使用中文输出
- 开发说明默认中文
- 新增注释默认中文
- 用户文案默认中文
- 代码标识符、接口字段、第三方固定参数保留英文

## 设计总原则

- 移动端优先
- 小程序优先
- 单手操作友好
- 快速扫读
- 低学习成本
- 稳定优先于花哨

## 样式总原则

- 如果项目已有品牌规范，优先继承项目现有体系
- 如果项目没有规范，再建立少量稳定的 token
- 颜色、圆角、间距、字号应先语义化，再组件化
- 同一项目里不要并存多套竞争关系很强的样式体系

## 页面总原则

- 页面结构要稳定
- 第一屏要让用户知道自己能做什么
- 主操作要容易发现
- 顶部、主体、底部操作区职责要清楚

## 反馈总原则

- 状态反馈及时
- 文案简短直接
- 先告诉用户发生了什么
- 必要时告诉用户下一步做什么
- 不把底层技术信息直接展示给用户

## 推荐 token 分层

推荐分为 4 层：
1. 基础值
2. 语义值
3. 组件值
4. 页面值

不要一开始就把所有颜色和尺寸散写进组件。

## 核心设计检查

进入一个新项目时，先看清这些东西：
- 有没有颜色、圆角、间距 token
- 有没有页面壳和导航模式
- 有没有统一的空态、加载态、错误态
- 有没有组件库或半成型组件层
- 有没有品牌文案口吻

如果这些都没有，再用本 skill 的风格方案作为默认基线。
