# 通用交互规范

这个文件定义小程序项目中普遍适用的交互规则。

## 交互原则

- 操作路径短
- 主操作清楚
- 状态反馈及时
- 用户少思考一步就少思考一步

## 页面状态

关键页面至少考虑：
- loading
- success
- empty
- error
- retry

如果只实现成功态，说明页面还没设计完整。

## 列表页

- 列表页通常要提供搜索、筛选、排序中的至少一种能力
- 单行优先展示最关键字段
- 空列表必须有空态
- 行内不要堆太多按钮
- 长列表要考虑性能和返回定位体验

## 表单页

- 字段按业务理解顺序排，不按接口顺序排
- 选择优先于输入
- 校验尽量贴近字段
- 主提交操作要稳定可见
- 键盘弹起不能挡住当前输入区和主按钮

## 详情页

- 第一屏先给结论、状态和关键摘要
- 相关信息按组展示
- 操作按优先级排序
- 不要把详情页做成字段墙

## 我的页 / 设置页

- 常用入口放前
- 设置项按分组展示
- 风险操作单独放后面
- 身份摘要和设置列表分区清楚

## 弹层与确认

- 能在当前页完成的，不要强制跳转
- 能轻量确认的，不要做重弹层
- 删除、解绑、退出、覆盖关键数据等行为需要二次确认
- 不要把所有操作都做成确认框

## 交互禁区

- 把 hover 作为核心依赖
- 用复杂手势替代普通点击
- 让用户在多个弹层里来回跳
- 一屏里放多个同等级主操作

## 最终检查

- 用户第一眼能否理解这个页面
- 主任务是否足够突出
- 失败时是否有清楚反馈
- 重复点击和弱网时是否仍然稳定
