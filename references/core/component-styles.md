# 组件样式契约

这个文件定义小程序项目里最常复用的几个核心组件的统一样式规则。

它关注的是视觉契约和状态契约，不替代组件边界文档。
组件是否该抽、接口怎么定，优先看 `component-boundary.md`。

## 适用范围

优先统一这些组件：

- `Button`
- `FormItem`
- `Card`
- `BottomActionBar`
- `Tag`
- `EmptyState`

如果项目还没到这个规模，至少先统一前四个。

## 总原则

- 组件样式优先依赖语义 token，不直接写原始色值
- 同类组件的高度、圆角、内边距和状态要稳定
- 组件状态比装饰更重要
- 页面优先复用组件，不要每个页面各画一套按钮和卡片

## Button

### 职责

- 承载明确动作
- 建立页面主次操作层级

### 最小变体

- `primary`
- `secondary`
- `ghost`
- `danger`

### 样式规则

- 默认高度统一，常规按钮优先使用同一高度
- 主按钮使用 `brand` 或品牌深色体系
- 主按钮文字必须有足够对比度
- 次按钮用柔和底色或描边，不与主按钮抢视觉重心
- 危险按钮必须和普通按钮有清楚语义差异

### 状态规则

- `default`
- `pressed`
- `disabled`
- `loading`

### 禁止事项

- 同一页面出现两个视觉同级的主按钮
- `loading` 时还能重复点击
- 仅靠颜色深浅区分危险动作

## FormItem

### 职责

- 承载标签、值区、辅助说明、错误提示
- 统一字段级布局和反馈位置

### 组成

- label
- value area
- helper text
- error text
- affordance，例如箭头、清除、语音入口

### 样式规则

- 标签位置稳定
- 值区高度和内边距统一
- 必填标记位置统一
- 错误提示与字段本体距离固定
- 同类字段的圆角、边框和背景处理保持一致

### 状态规则

- `default`
- `focused`
- `filled`
- `error`
- `disabled`

### 禁止事项

- 让占位文案代替字段标签
- 错误提示漂在离字段很远的位置
- 文本输入、picker、语音输入各用一套外壳

## Card

### 职责

- 承载分组信息
- 建立页面内容层次

### 样式规则

- 默认使用统一背景色
- 圆角和内边距固定在少数几个档位
- 阴影尽量轻，工具型页面优先靠层级和留白区分
- 卡片标题、摘要、操作区的间距关系要稳定

### 禁止事项

- 同一页面同时出现多套卡片圆角和阴影体系
- 把所有内容都塞进无层次的大白块

## BottomActionBar

### 职责

- 承载页面最后一步关键动作
- 在滚动和键盘场景下保持操作可达

### 样式规则

- 要考虑安全区
- 默认与页面主体拉开层级
- 内边距和按钮间距固定
- 背景、分割线或阴影要稳定，不要一页一套

### 状态规则

- `default`
- `disabled`
- `loading`

### 禁止事项

- 遮挡主体内容和错误提示
- 键盘弹起时把主按钮挤出可视区域
- 同时放过多次要操作

## Tag

### 职责

- 展示轻量状态和分类

### 样式规则

- 高度和左右 padding 统一
- 颜色优先走语义状态，不用随机装饰色
- 同一页面标签数量多时，优先弱化装饰，保留识别度

## EmptyState

### 职责

- 说明当前没有数据或结果
- 在必要时提供下一步动作

### 组成

- 图形或图标
- 现状说明
- 可选动作按钮

### 样式规则

- 图形风格稳定
- 主文案简短直接
- 若有动作，通常只保留一个主动作

## 自检

- 这些组件是否仍然各页各写一套
- 是否已经从语义 token 取值
- 高度、圆角、边框、间距是否稳定
- loading、disabled、error 是否都有统一处理
- 页面是否通过复用组件建立了真正的一致性
