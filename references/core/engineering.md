# 工程实现约束

这个文件定义适合多数小程序项目的工程实现基线。

## 平台意识

- 默认把小程序当成主要运行环境
- 不假设浏览器专属能力一定存在
- 不假设桌面宽屏和鼠标交互
- 固定底部区、弹层、键盘、滚动容器属于高风险区域

## 实现原则

- 优先复用现有组件和基础设施
- 优先写清楚，再谈抽象
- 优先稳定可维护，再谈技巧
- 优先产品语义，再谈技术实现细节

## 架构检查

进入新项目时，先找清楚这些层：
- 页面层
- 组件层
- 状态层
- 请求层
- 错误处理层

如果项目已经分层清楚，优先沿用。

## 样式约束

- 项目已有 token 时优先复用 token
- 项目没有 token 时再建立少量稳定变量
- 不要在一个项目里混入多套竞争关系很强的样式体系
- 局部内联样式可以接受，但重复模式应及时收敛

## 错误处理

- 对用户：给可理解、可执行的中文提示
- 对开发：在日志或调试层保留足够信息
- 不在界面里直接展示底层报错原文

## 性能与稳定性

- 谨慎处理长列表
- 谨慎处理频繁重渲染
- 谨慎处理多层滚动
- 谨慎处理重复请求和重复点击
